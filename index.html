什么是面向对象

面向对象是一种软件开发的编程方式，与面向过程是两种不同的编程方式

特点

面向过程：所有功能都是在需要使用的时候开发

面向对象：在正式开发之前，就先把所有需要使用的功能都开发好，并以属性或者方法的形式存放在一个对象中，在实际开发的时候根据需要来调用写好的功能

JavaScript是最不像面向对象的面向对象语言，因为JavaScript中没有类的概念，也不能像传统面向对象语言那样去实现封装、继承和多态。

面向对象模式

面向对象的工厂模式

function person(name,age){
    var obj={};
    obj.name=name;
    obj.age=age;
    obj.say=function(){
        alert('我叫'+this.name+'，我今年'+this.age+'岁。');
    };
    return obj;
}
var xiaoming=person('小明',18);
var xiaohong=person('小红',16);
xiaoming.say();
xiaohong.say();

1.工厂方式写法与面向过程写法相同，相对比较好了解

2.工厂方式的缺点在于所有属性和方法都会被重复创建

####面向对象的构造函数模式
function Person(name,age){
this.name = name;
this.age = age;
this.say = function(){
alert(‘我叫’+this.name+’,我今年’+this.age+’岁。’)；
};
}
var xiaoming = new Person(‘小明’,18);
var xiaohong = new Person(‘小红’,16);
xiaoming.say();
xiaohong.say();

构造函数：用来构建对象的函数

1.为了区别普通函数，构造函数的函数名首字母规定应该为大写。

2.构造函数必须使用new运算符来调用执行。

普通函数与构造函数中的this。

普通函数的this与调用时的上下文环境有关。

构造函数中的this为一个空的对象，并且会作为return的默认值。

JavaScript内置提供的构造函数

String() / Array() / Date() / Number() / Boolean() / Object() / Function() / RegExp()

构造函数+原型模式

function Person(name,age){
    this.name=name;
    this.age=age;
}
Person.prototype.say=function(){
    alert('我叫'+this.name+',我今年'+this.age+'岁。');
};
var xiaoming=new Person('小明',18);
var xioahong=new Person('小红',16);
xiaoming.say();
xiaohong.say();

原型是函数的一个特有属性。

1.原型本身是一个对象。

2.原型对象上的属性和方法可以被实例对象使用。

普通对象中的属性与原型中的属性的区别

1.普通对象的属性：只能在拥有属性的对象中使用。

2.原型对象的属性：所有通过同一个构造函数创建的对象中都可以使用。

原型链：原型链是指对象在访问属性或方法时的查找方式。

1.当访问一个对象的属性或方法时，会先在对象自身上查找属性或方法是否存在，如果存在就使用对象自身的属性或方法。如果不存在就去创建对象的构造函数的原型对象中查找，以此类推，直到找到为止。如果到顶层对象中还找不到，则返回undefined.

2.原型链最顶层为Object构造函数的prototype原型对象，给Object.prototype添加属性或方法可以被除null和undefined之外的所有数据类型对象使用。

constructor与instanceof

constructor属性(举例:nodeType)

通过constructor属性可以查看一个实例对象是由哪一个构造函数创建出来的。

  1. constructor是构造函数的prototype中的属性,它的值为构造函数本身

  2. constructor属性可以被修改

instanceof运算符

instanceof运算符用来查看一个实例对象是不是由某个构造函数创建出来的

####1
function Foo() {
getName = function () { alert (1); };
return this;
}
Foo.getName = function () { alert (2);};
Foo.prototype.getName = function () { alert (3);};
var getName = function () { alert (4);};
function getName() { alert (5);}

//请写出以下输出结果
1.Foo.getName();
2.getName();
3.Foo().getName();
4.getName();
5.new Foo.getName();
6.new Foo().getName();
7.new new Foo().getName();

答案是：

1. 2 
2. 4
3. 1
4. 1
5. 2
6. 3
7. 3

详细解析
先将这几个函数按优先级排列

function Foo(){
    getName = function (){ alert (1); };
    return this;
};

function getName(){
    alert(5);
};

var getName;

Foo.getName = function () { 
    alert (2);
};

Foo.prototype.getName = function () { 
    alert (3);
};

getName = function() { 
    alert (5);
}

1.第一问的 Foo.getName 自然是访问Foo函数上存储的静态属性，自然是2

2.直接调用getName函数，而var getName = function(){ alert(4); }中的变量声明提升,
得出“var getName;getName=function(){ alert(4); }”,后面的覆盖前面的getName函数，故此题为4

3.先将Foo()函数运行，其中的return值为this，this指向window,所以第3题就等于window.getName(),Foo函数运行后其中的getName函数在全局中出现，放在最后，所以此题弹出值为1

4.直接调用getName函数，与上题相同，弹出1

5.new Foo.getName(),此处考察的是js运算符优先级的问题，.的运算级高于new操作，于是为new（Foo.getName）(),所以弹出2

6.new Foo().getName(),运算符优先级（）高于new，所以为（new.Foo()）.getName()

网格系统

实现原理：

网格系统的实现原理非常简单，仅仅是通过定义容器大小，平分12份(也有平分成24份或32份，但12份是最常见的)，再调整内外边距，最后结合媒体查询，就制作出了强大的响应式网格系统。Bootstrap框架中的网格系统就是将容器平分成12份

工作原理：

数据行(.row)必须包含在容器（.container）中，以便为其赋予合适的对齐方式和内距(padding)

在行(.row)中可以添加列(.column)，但列数之和不能超过平分的总列数，比如12

具体内容应当放置在列容器（column）之内，而且只有列（column）才可以作为行容器(.row)的直接子元素

通过设置内距（padding）从而创建列与列之间的间距。然后通过为第一列和最后一列设置负值的外距（margin）来抵消内距(padding)的影响

npm的发布

1.符合规范的目录需求：

package.json

README.md(可选的)

2.npm 注册：npm adduser

3.初始化npm目录：npm init

4.目标模块上传：npm publish 目标模块删除（已上传）：npm unpublish
